# Rubric 评分维度定义

本文档定义了 Skill Auditor 使用的完整评分体系。

---

## 评分维度

### 1. 格式规范 (15%)

评估技能是否符合 Agent Skills 规范结构。

| 分数区间 | 标准 |
|----------|------|
| 90-100 | 完美符合规范，无任何问题 |
| 70-89 | 符合规范，有少量 WARNING 级别问题 |
| 50-69 | 基本符合，有 ERROR 级别问题需修复 |
| 0-49 | 严重不符合，有 CRITICAL 问题 |

**检查项**:
- [ ] SKILL.md 文件存在
- [ ] YAML frontmatter 格式正确
- [ ] `name` 字段符合 kebab-case 格式
- [ ] `name` 与目录名一致
- [ ] `description` 长度 20-1024 字符
- [ ] `description` 包含使用场景说明
- [ ] SKILL.md 行数 < 500
- [ ] 所有引用文件存在
- [ ] 脚本有入口点

**不参与扣分的可选字段**：`homepage`、`author`、`disable-model-invocation` 等非必填字段缺失不扣分。审计报告中**禁止建议填入无法验证的外部地址**（URL、仓库链接等），避免诱导编造值。

---

### 2. 安全性 (25%)

检测潜在安全风险，权重最高。

| 分数区间 | 标准 |
|----------|------|
| 100 | 无任何安全问题 |
| 75-99 | 有 LOW/MEDIUM 级别问题 |
| 25-74 | 有 HIGH 级别问题 |
| 0-24 | 有 CRITICAL 级别问题 |

**检查类别**:

| 类别 | 扣分 | 检测内容 |
|------|------|----------|
| Prompt Injection | -25/个 | 隐藏指令、角色覆盖、jailbreak |
| 秘钥泄露 | -25/个 | API keys, tokens, passwords |
| 危险代码 | -15/个 | eval(), exec(), subprocess shell=True |
| 数据外泄 | -15/个 | 未经验证的外部请求 |
| 权限过宽 | -5/个 | 不必要的工具权限 |

---

### 3. 文档质量 (20%)

评估技能文档的可读性和完整性。

| 子维度 | 权重 | 评估标准 |
|--------|------|----------|
| 可读性 | 15% | 句子长度适中(10-20词)，有代码示例 |
| 清晰性 | 20% | 使用动作动词，避免"you should" |
| 完整性 | 25% | 覆盖必要章节，有足够示例 |
| 一致性 | 15% | 标题层级、列表格式统一 |
| 文档化 | 15% | 脚本有 docstring |
| 实用性 | 10% | 有可执行命令示例 |

**必要章节**:
- Overview/概述
- When to Use/使用场景
- Instructions/操作步骤
- Examples/示例

---

### 4. 能力覆盖 (15%)

分析技能的功能范围和触发场景。

| 分数区间 | 覆盖度估计 |
|----------|------------|
| 90-100 | 95%+ 全面覆盖 |
| 75-89 | 85-95% 高覆盖 |
| 60-74 | 70-85% 中等覆盖 |
| 40-59 | 50-70% 基础覆盖 |
| 0-39 | <50% 有限覆盖 |

**评估因素**:
- 触发关键词数量和质量
- 功能列表完整性
- 使用场景多样性
- 依赖资源丰富度
- 复杂度评分(1-10)

---

### 5. 可维护性 (15%)

评估技能的长期维护成本。

| 分数区间 | 标准 |
|----------|------|
| 85-100 | 模块化良好，有参考文档分离 |
| 70-84 | 结构清晰，脚本组织合理 |
| 50-69 | 基本可维护，有改进空间 |
| 0-49 | 结构混乱，维护困难 |

**加分项**:
- 有 references/ 目录 (+3/文档)
- 脚本有子目录组织 (+5)
- 有 __init__.py 模块化 (+5)
- SKILL.md < 400 行 (+5)

**减分项**:
- SKILL.md > 500 行 (-15)
- 有重复内容 (-10)
- 无参考文档分离 (-5)

---

### 6. 实用性 (10%)

评估技能的实际可用性。

| 分数区间 | 标准 |
|----------|------|
| 85-100 | 有多个可执行示例和完整脚本 |
| 70-84 | 有代码示例和基本脚本 |
| 50-69 | 仅有文档描述 |
| 0-49 | 缺少实用内容 |

**加分项**:
- 代码示例 >= 3 个 (+10)
- 有可执行脚本 (+4/个，最多+10)
- 有 CLI 接口 (+5)
- 有资源模板 (+5)

---

## 综合等级

| 等级 | 分数区间 | 星级 | 说明 |
|------|----------|------|------|
| A+ | 90-100 | ⭐⭐⭐⭐⭐ | 卓越 - 可作为参考模板 |
| A | 80-89 | ⭐⭐⭐⭐ | 优秀 - 生产就绪 |
| B | 70-79 | ⭐⭐⭐ | 良好 - 可用但有改进空间 |
| C | 60-69 | ⭐⭐ | 合格 - 需要改进 |
| D | 0-59 | ⭐ | 不合格 - 需要重大修改 |

## 自定义配置

可以通过 JSON 配置文件自定义评分权重：

```json
{
  "dimensions": {
    "format": {"weight": 0.15},
    "security": {"weight": 0.25},
    "quality": {"weight": 0.20},
    "capability": {"weight": 0.15},
    "maintainability": {"weight": 0.15},
    "practicality": {"weight": 0.10}
  },
  "grades": {
    "A+": {"min": 90},
    "A": {"min": 80},
    "B": {"min": 70},
    "C": {"min": 60},
    "D": {"min": 0}
  }
}
```

使用自定义配置：
```bash
python scripts/scorers/rubric_scorer.py ./my-skill --config custom_rubric.json
```
