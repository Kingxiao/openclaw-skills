# 设计模式：客户管理（CRM）

> verified: 2026-04-16

适用场景：用户说"做一个客户管理系统 / CRM / 销售管理 / 客户跟进"。

## 推荐表结构（4 张表）

### 1. 客户表（核心）
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户名称 | text | ✅ | 主字段 |
| 客户类型 | single_select | | 企业 / 个人 / 渠道 |
| 状态 | single_select | ✅ | 进行中 / 已签约 / 流失 / 暂停 |
| 负责人 | user | ✅ | 销售 owner |
| 来源 | single_select | | 自来 / 转介绍 / 营销活动 / 其他 |
| 行业 | single_select | | 按业务问 |
| 主联系人 | text | | 简版，详细联系人走联系人表 |
| 主电话 | phone | | |
| 创建时间 | created_time | | 系统字段 |
| 最近跟进 | datetime | | 手动维护或公式取最大跟进记录时间 |
| 备注 | text | | |

### 2. 联系人表（客户的联系人，1:N）
| 字段 | 类型 | 必填 |
|------|------|------|

| 微信 | text | |

| 字段 | 类型 | 必填 |

## 推荐关系

- 客户 1:N 联系人（删除客户保护——必须先处理联系人）
- 客户 1:N 跟进记录（同上）
- 客户 1:N 订单（同上）
- 联系人 1:N 跟进记录（可选关联）

## 推荐视图

**客户表**：
- 客户总览（默认）：所有状态非"流失"的客户，按"最近跟进"倒序
- 我的客户：filter `负责人 == ME`
- 待跟进预警：filter `最近跟进 < 7天前 AND 状态 == 进行中`
- 已签约：filter `状态 == 已签约`
- 流失客户：filter `状态 == 流失`，按时间分组

**跟进记录表**：
- 本周跟进（按跟进时间 7 天内）
- 我的跟进
- 按客户分组

## 推荐自动化

1. **新客户提醒**：客户表新增 → 通知负责人
2. **跟进逾期预警**：每天检查 `最近跟进 < 7天前 AND 状态 == 进行中` 的客户 → 推送给负责人
3. **签约通知**：客户状态变 `已签约` → 通知销售经理（群消息）
4. **客户失联预警**：30 天无跟进 → 标记为"流失风险"

## 推荐仪表盘

**销售看板**：
- 本月新增客户数（数字卡片）
- 客户状态分布（饼图）
- 销售员客户数 TOP 10（柱状图）
- 待跟进客户清单（列表）
- 本月签约金额（数字 + 趋势）

## 推荐角色

- **销售**：只能看自己负责的客户、对应联系人/跟进记录/订单（基于 `负责人 == ME` 过滤）
- **销售经理**：看本团队（按部门字段过滤，需补充字段）
- **管理员**：全部权限

## 推荐表单

**客户登记表单**（分享给营销活动用）：
- 客户名称 / 主联系人 / 电话 / 来源 / 行业
- 自动设当前提交人为负责人
- 提交后 → 自动化通知销售经理分配

## 业务追问的热点

按 [business_questioning_guide.md](../business_questioning_guide.md) 的模板，CRM 场景常追问：
1. 客户状态都有哪些？是否需要"暂停"状态？
2. 一个客户能有多个负责人吗？（决定字段类型 single user vs multi user）
3. 销售要看到别人的客户吗？看到时能改吗？
4. 跟进记录的"下次跟进计划"逾期要提醒谁？
5. 订单退款的处理流程？
