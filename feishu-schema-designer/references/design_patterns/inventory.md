# 设计模式：库存管理

> verified: 2026-04-16

适用场景：用户说"做一个库存管理 / 仓储 / 商品管理 / 进销存 / SKU 管理"。

## 推荐表结构（5-6 张表）

### 1. SKU 表（商品主数据）
| 字段 | 类型 | 必填 |
|------|------|------|
| SKU 编号 | text 或 auto_number | ✅ |
| 商品名称 | text | ✅ |
| 类别 | single_select | 自定义类别 |
| 规格 | text | |
| 单位 | single_select | 件 / 箱 / 套 / 公斤 / 米 |
| 单价 | currency | |
| 安全库存 | number | 阈值，低于触发预警 |
| 当前库存 | formula 或 number | 公式 = SUM(入库) - SUM(出库)；或手动维护 |
| 启用状态 | checkbox | 默认勾选 |
| 备注 | text | |

### 2. 仓库表
| 字段 | 类型 | 必填 |
|------|------|------|
| 仓库名称 | text | ✅ |
| 地址 | text | |
| 负责人 | user | ✅ |
| 备注 | text | |

### 3. 出入库记录表（核心台账）
| 字段 | 类型 | 必填 |
|------|------|------|
| 单号 | auto_number | ✅ |
| 类型 | single_select | 入库 / 出库 / 调拨 / 盘盈 / 盘亏 |
| SKU | single_link → SKU | ✅ |
| 仓库 | single_link → 仓库 | ✅ |
| 数量 | number | ✅（出库用负数 OR 单独"方向"字段+正数） |
| 单价 | currency | 入库时记录采购成本 |
| 总价 | formula | = 数量 × 单价 |
| 经办人 | user | ✅ |
| 业务时间 | datetime | ✅ |
| 关联订单 | text | 可选关联外部销售订单号 |
| 备注 | text | |

### 4. 库存快照表（按需，定期盘点）
| 字段 | 类型 | 必填 |
|------|------|------|
| 盘点编号 | auto_number | ✅ |
| 盘点日期 | datetime | ✅ |
| SKU | single_link → SKU | ✅ |
| 仓库 | single_link → 仓库 | ✅ |
| 系统库存 | number | 盘点前系统数 |
| 实盘库存 | number | ✅ |
| 差异 | formula | = 实盘 - 系统 |
| 处理 | single_select | 已调整 / 待处理 |
| 盘点人 | user | ✅ |

### 5. 供应商表（按需）
| 字段 | 类型 | 必填 |
|------|------|------|
| 供应商名称 | text | ✅ |
| 联系人 | text | |
| 电话 | phone | |
| 主营 SKU | single_link → SKU（多选） | |
| 信用等级 | single_select | A / B / C |

### 6. 预警表（可选，自动化产物，agent 不必先建）
若用 workflow 直接推送预警，不建表也行；若想留预警历史，再建。

## 推荐关系

- SKU 1:N 出入库记录
- 仓库 1:N 出入库记录
- SKU 1:N 库存快照
- SKU N:N 供应商

## 推荐视图

**SKU 表**：
- 默认（启用 SKU）：filter `启用状态 == true`
- 库存预警：filter `当前库存 < 安全库存`
- 按类别分组

**出入库记录表**：
- 今日台账（filter `业务时间 == today`）
- 我的台账（按经办人 == ME）
- 入库 / 出库（按类型分视图）
- 按 SKU 分组的近 30 天

## 推荐自动化

1. **库存预警**：定时（每日早晨）扫描 `当前库存 < 安全库存` 的 SKU → 推送给采购负责人
2. **大额出库提醒**：出库记录数量 × 单价 > 阈值 → 通知主管
3. **盘亏处理提醒**：库存快照差异 < 0 → 通知仓库负责人 + 财务

## 推荐仪表盘

**库存看板**：
- 总 SKU 数 / 启用 SKU 数（数字）
- 总库存价值（数字，formula = SUM(SKU 当前库存 × 单价)）
- 库存预警 SKU 数（数字 + 列表）
- 本月入库 / 出库金额（趋势图）
- 各仓库库存分布（柱状图）
- TOP 10 高频 SKU（按出入库次数）

## 推荐角色

- **仓管**：只能看自己负责的仓库的台账（filter `仓库.负责人 == ME`）
- **采购**：看全部 SKU + 入库记录
- **管理员**：全部

## 业务追问热点

1. 多仓库还是单仓库？（决定是否建仓库表）
2. 需要批次管理吗？（一个 SKU 多个批次有不同保质期/批号）→ 加批次表
3. 出库数量用负数还是 +"方向"字段？（前者公式简单，后者更直观）
4. 需要供应商管理吗？
5. 盘点频率？是否需要快照表保留历史？
6. 库存预警的安全阈值谁定？要不要按 SKU 类别有不同阈值？
7. 出库是否需要审批流？（如果是，加 lark-approval 联动）

## 复杂扩展（按需）

- 批次/保质期管理 → 在 SKU 和出入库间加批次表
- 多仓库调拨 → 出入库记录的"类型 == 调拨"，需要同时加一进一出（自动化触发）
- 采购订单 → 单独建采购单表（采购单 1:N 入库记录）
- 销售订单 → 同上，销售单 1:N 出库记录
