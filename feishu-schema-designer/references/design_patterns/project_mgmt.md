# 设计模式：项目管理

> verified: 2026-04-16

适用场景：用户说"做一个项目管理系统 / 项目跟踪 / 任务管理 / 工作进度"。

## 推荐表结构（5 张表）

### 1. 项目表
| 字段 | 类型 | 必填 |
|------|------|------|
| 项目名称 | text | ✅ |
| 项目代号 | text | |
| 状态 | single_select | 规划中 / 进行中 / 已完成 / 已暂停 / 已取消 |
| 负责人 | user | ✅ |
| 团队成员 | user (多选) | |
| 开始日期 | datetime | |
| 计划完成日期 | datetime | |
| 实际完成日期 | datetime | |
| 优先级 | single_select | P0 / P1 / P2 / P3 |
| 项目类别 | single_select | 内部 / 客户项目 / 实验性 |
| 总预算 | currency | |
| 备注 | text | |
| 进度 | progress | 也可由公式计算 = 已完成任务数/总任务数 |

### 2. 任务表

| 父任务 | single_link → 任务（可选，做子任务） | |
| 状态 | single_select | 待开始 / 进行中 / 已完成 / 阻塞 / 取消 |
| 负责人 | user | ✅ |
| 协作者 | user (多选) | |
| 优先级 | single_select | P0 / P1 / P2 / P3 |
| 计划开始 | datetime | |
| 计划完成 | datetime | |
| 实际完成 | datetime | |
| 工时（小时） | number | |

### 3. 里程碑表
| 字段 | 类型 | 必填 |
|------|------|------|

## 推荐关系

- 项目 1:N 任务（删项目保护）
- 任务 1:N 子任务（自引用）
- 项目 1:N 里程碑
- 里程碑 N:N 任务
- 任务 1:N 工时记录
- 项目 1:N 问题

## 推荐视图

**项目表**：
- 进行中项目（默认，filter `状态 == 进行中`）
- 我负责的项目
- 看板视图（按状态分组）
- 时间轴视图（开始-完成日期，timebar 视图）

**任务表**：
- 我的待办（filter `负责人 == ME AND 状态 IN [待开始, 进行中]`，按优先级 + 计划完成排序）
- 看板视图（按状态分组）
- 本周到期（filter `计划完成 < 7 天后`）
- 已逾期（filter `计划完成 < 今天 AND 状态 != 已完成`）

**里程碑表**：
- 时间轴视图

## 推荐自动化

1. **任务分配通知**：任务表新增 → 通知负责人
2. **任务到期前 1 天预警**：每天扫描计划完成日期 → 推送
3. **任务逾期预警**：每天扫描已逾期 → 推送负责人 + 项目负责人
4. **项目状态变化通知**：项目状态变成"已完成"或"已暂停" → 通知团队成员
5. **里程碑达成庆祝**：里程碑状态变"已达成" → 群消息

## 推荐仪表盘

**项目看板**：
- 进行中项目数（数字）
- 本周完成任务数（数字 + 趋势）
- 项目状态分布（饼图）
- 各负责人任务量 TOP（柱状图）
- 逾期任务清单（列表）
- 里程碑日历（时间轴）

## 推荐角色

- **成员**：只能看自己参与的项目（filter `负责人 == ME OR ME IN 团队成员 OR ME IN 协作者`）
- **项目经理**：看自己负责的项目 + 全部任务
- **管理员**：全部

## 业务追问热点

1. 是否需要支持子任务？（决定任务表自引用）
2. 工时记录是必需还是可选？（影响是否建工时表）
3. 任务优先级用 P0-P3 还是高/中/低？
4. 里程碑达成的判定标准？人工标记还是自动（关联任务全部完成）？
5. 谁能看其他人的项目？
6. 跨项目的资源冲突如何检测？
